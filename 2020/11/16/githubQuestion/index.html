浏览器打不开github pages和无法从github下载图片

浏览器打不开github pages和无法从github下载图片，这是因为自己电脑的DNS有问题

这里介绍两种方法：一种是直接修改DNS为：223.5.5.5和223.6.6.6（阿里的DNS),另一种是修改hosts文件下面是第二种方法的介绍

首先打开这个ip查询网站查询网，输入raw.githubusercontent.com。

复制 中国香港的ip地址 ：151.101.76.133（以后可能会变化）

在本机中，打开C:\Windows\System32\drivers\etc\hosts，打开hosts文件把下面的代码加入hosts并保存即可。

如果不起作用，win+r快捷键调出CMD 输入ipconifg /flushdns 就可以解决上面两个问题了

其实我在修改hosts时还遇到一个问题，就是没有修改权限。解决办法如下

如何修改文件权限

  1. 选中需要修改权限的文件夹或文件，右键选择属性
  2. d点击安全，点击高级。
  3. 点击添加 ，然后选择主题
  4. 在点击高级 ，点击立即查找 会有许多结果。选择第二个admin 然后确定。回到选择主题，选项卡，把所有权限都勾选。让后依次确认即可。
  5. 之后就可以修改hosts了。

使用git bash 添加文件夹时 显示modified: (modified content, untracked content)

报错modified: 文件夹名 (modified content, untracked content)

这是因为在这个文件夹下已经有个git文件夹了。把这个文件夹删除即可（git文件夹是被隐藏的）

git 安装后环境配置

  1. git 安装好后，需要为这台电脑配置注册账号的git用户名和git邮箱。
# 配置用户名
git config --global user.name "username"    //（ "username"是自己的账户名，）
# 配置邮箱
git config --global user.email "username@email.com"     //("username@email.com"注册账号时用的邮箱)

以上命令执行结束后，可用 git config –global –list 命令查看配置是否OK

  1. 生成ssh

    继续刚才的操作，在命令框中输入以下命令

ssh-keygen -t rsa
然后连敲三次回车键，结束后去系统盘目录下（一般在 C:\Users\你的用户名.ssh）(mac: /Users/用户/.ssh）查看是否有。ssh文件夹生成，此文件夹中以下两个文件

id_rsa和id_rsa.pub

  1. 用记事本打开id_rsa.pub复制其中的内容，登录github，点击右上角的github头像，然后点击setting，进入账户设置。

  2. 点击SSH and GPG keys ，然后点击NEW SSH KEY

  3. title随便起一个，将公钥（ id_rsa.pub）文件中内容复制粘贴到key中，然后点击Ass SSH key就行了

  4. 测试一下配置是否成功，在Git Bush命令框（就是刚才配置账号和邮箱的命令框）中继续输入以下命令，回车

ssh -T git@github.com 如果出现 You've successful...等文字就成功了

GitHub Pages自定义域名后每次hexo d都会失效解决

只要在source 目录添加一个新文件CNAME就好

CNAME 不带任何后缀，这就是全称，里面写的是你的域名

git 不能clone 显示拒绝连接

  1. 查看是否有代理被占用

git config --global http.proxy

如果显示：

就是被占用了

  1. 取消代理

git config --global --unset http.proxy
